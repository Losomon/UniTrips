# Security Policy

## Reporting Security Issues

**Thank you for improving the security of UniTrips!** 🔒

### Preferred Method
Report vulnerabilities by opening a GitHub issue or emailing the repository maintainer directly.

### What to include
```
- Type of issue (XSS, injection, etc.)
- Full repro steps
- Impact assessment
- Affected versions
- Proposed fix (optional)
```

### Supported Guidelines
- [GitHub Advisory Database](https://github.com/advisories)
- [OSV Format](https://osv.dev/docs/)

## Response SLA
- **Critical**: 24h ACK, 72h fix
- **High**: 48h ACK, 7d fix
- **Medium/Low**: 1w ACK, 30d fix

## Scope
- Frontend SPA (index.html/js/css)
- Backend JSP app
- No third-party deps (pure vanilla)

## Disclosed Vulnerabilities
None currently.

## Rewards
Researcher credits in changelog for valid reports.

---

**Safe harbor**: No legal action for good-faith reports.

Thanks for keeping UniTrips secure! 🛡️
